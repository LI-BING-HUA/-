### 🔴 Bcdadd100 — BCD 100 位加法器

| 項目 | 內容 |
|------|------|
| 難度 | 🔴 |
| 首刷日 | 2026/05/18 |
| 複習 1 | ☐ 2026/05/21 |
| 複習 2 | ☐ 2026/05/25 |
| 複習 3 | ☐ 2026/06/08 |
| 狀態 | 🔴 學習中 |

**卡點**
- 以為 BCD 就是普通二進位 → 錯!BCD 每位只能 0~9
- `a[i*4 +: 4]` 切片語法不會讀
- 不懂為什麼 `bcd_fadd` 要做「修正」

**頓悟點(複習時先遮住)**
- BCD `7+5`:純二進位 = `1100`(12),但 BCD 不能輸出 12 → 修正成 `sum=0010`(2)、`cout=1`,跟小學「寫 2 進 1」一樣
- 進位條件是「**超過 9**」,不是「4-bit 用盡(超過 15)」
- `a[i*4 +: 4]` = 從 bit `i*4` 開始往上取 4 個 bit(因為 Verilog 不准 `a[變數:變數]`,所以用 `+:` 起點可變、寬度固定)
- `bcd_fadd` 是黑盒子,修正邏輯它處理好了,我只要串接力

**模板(複習時先遮住,自己推)**
```verilog
module top_module (
    input  [399:0] a, b,
    input          cin,
    output         cout,
    output [399:0] sum
);
    wire [99:0] carry;
    genvar i;
    generate
        for (i = 0; i < 100; i = i + 1) begin : bcd_loop
            if (i == 0)
                bcd_fadd fa (.a(a[3:0]), .b(b[3:0]),
                             .cin(cin), .cout(carry[0]), .sum(sum[3:0]));
            else
                bcd_fadd fa (.a(a[i*4 +: 4]), .b(b[i*4 +: 4]),
                             .cin(carry[i-1]), .cout(carry[i]), .sum(sum[i*4 +: 4]));
        end
    endgenerate
    assign cout = carry[99];
endmodule
```

## 📌 觀念速查(反覆踩雷的重點,隨時補充)

> 不是某一題,而是「**每次都會忘**」的通用觀念,集中記在這

- **`<=` vs `=`**:時序邏輯(有 clk)用 `<=`,組合邏輯(`always @(*)`)用 `=`
- **latch 怎麼來**:`always @(*)` 有路徑沒賦值 → latch;時序邏輯沒 else 是正常的(保持原值=FF)
- **避免 latch**:`always @(*)` 進 case 前先給所有 output 預設值
- **`casez` vs `casex`**:用 `casez` + `?`,永遠別用 `casex`(訊號是 x 時會誤匹配第一條)
- **part-select**:`a[i*4 +: 4]` = 起點 `i*4` 往上取 4 個(變數索引只能這樣寫)
- **reduction 運算子**:`&a` 全 1、`|a` 不為 0、`^a` 奇偶性(一行處理整個向量)
- **1-bit 加法**:`out = out + in[i]` 比 `if(in[i]) out=out+1` 簡潔(Verilog 把 1-bit 當數值)
- **實例化 vs assign**:實例化本身就是有效電路;每條 wire 要恰好一個 driver
- **named connection**:實例化永遠用 `.port(signal)`,別用按順序的 positional
- **tb 型別**:testbench 的 input 用 `reg`(自己驅動),output 用 `wire`(被 DUT 驅動)
